Check Virtual Machine Encryption

It is a function that checks whether each EC2 instance in an AWS environment has an encrypted root volume. It does this by first getting a list of all EC2 instances using the describe_instances() method of the ec2_client object. Then, for each instance, it checks the BlockDeviceMappings attribute to find the instance's root volume, and checks whether the Encrypted attribute of the volume is set to True. If any instance is found that does not have an encrypted root volume, the function prints a message to the console indicating the instance ID and the issue.
Check Virtual Machine Acess

A function that checks whether any security groups associated with each EC2 instance in an AWS environment have unrestricted access. It does this by first getting a list of all EC2 instances using the describe_instances() method of the ec2_client object. Then, for each instance, it retrieves the instance's security group IDs and checks each security group using the describe_security_group_rules() method of the ec2_client object. For each security group, it checks whether any rule allows unrestricted access by looking for a rule with -1 as the IpProtocol (which indicates all protocols), an empty list of PrefixListIds, and an empty list of UserIdGroupPairs. If any instance is found that has a security group with unrestricted access, the function prints a message to the console indicating the instance ID and the issue.
Check Multi-Factor Authentication Enabled

A function that checks whether multi-factor authentication (MFA) is enabled for all IAM users in an AWS environment. It does this by first getting a list of all IAM users using the list_users() method of the iam_client object. Then, for each user, it checks whether they have any MFA devices associated with their account using the list_mfa_devices() method of the iam_client object. If no MFA devices are found for a user, the function prints a message to the console indicating the user's name and the issue.
Check Logging

A function that checks whether logging is enabled for all CloudTrail trails in an AWS environment. It does this by first getting a list of all trails using the describe_trails() method of the cloudtrail_client object. Then, for each trail, it checks whether logging is enabled by calling the get_trail_status() method of the cloudtrail_client object and examining the IsLogging field in the response. If logging is not enabled for a trail, the function prints a message to the console indicating the trail's name and the issue.
Check Patches

A function that checks whether there are any pending patches available for EC2 instances in an AWS environment. It does this by first getting a list of all instances using the describe_instances() method of the ec2_client object. Then, for each instance, it calls the describe_instance_patches() method of the ec2_client object to get information about any available patches. If there are any pending patches for an instance, the function prints a message to the console indicating the instance ID and the number of pending patches.
Check Unused Resources

A function that checks whether there are any EC2 instances in an AWS environment that have not been used within the past 30 days. It does this by first getting a list of all instances using the describe_instances() method of the ec2_client object. Then, for each instance, it calls the lookup_events() method of the cloudtrail_client object to get a list of all CloudTrail events associated with the instance within the past 30 days. If there are no events for the instance within the past 30 days, the function prints a message to the console indicating the instance ID and the issue. The function uses datetime module to set the start time and end time for the CloudTrail event lookup.
